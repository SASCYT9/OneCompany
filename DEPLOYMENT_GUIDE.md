# Інструкція з налаштування та деплою OneCompany

Цей документ містить детальні інструкції щодо налаштування домену, пошти, редагування текстів та розгортання (деплою) сайту.

## 1. Підв'язка домену з GoDaddy до Vercel

Щоб ваш сайт відкривався за вашим доменом (наприклад, `onecompany.global`), потрібно налаштувати DNS записи.

### Крок 1: Додавання домену у Vercel
1. Зайдіть у свій акаунт на [Vercel Dashboard](https://vercel.com/dashboard).
2. Виберіть проєкт **OneCompany**.
3. Перейдіть у вкладку **Settings** (Налаштування) -> **Domains** (Домени).
4. Введіть ваш домен (наприклад, `onecompany.global`) у поле і натисніть **Add**.
5. Vercel покаже вам необхідні DNS записи. Зазвичай це:
   - **A Record**: `@` -> `76.76.21.21`
   - **CNAME Record**: `www` -> `cname.vercel-dns.com`

### Крок 2: Налаштування DNS на GoDaddy
1. Зайдіть у свій акаунт [GoDaddy](https://godaddy.com).
2. Перейдіть до списку ваших доменів і натисніть **DNS** або **Manage DNS** біля потрібного домену.
3. У розділі **DNS Records** (DNS Записи) додайте або змініть наступні записи (відповідно до того, що показав Vercel):
   - **Type**: `A`
   - **Name**: `@`
   - **Value**: `76.76.21.21` (або IP, який дав Vercel)
   - **TTL**: `600` (або `1 Hour`)
   
   - **Type**: `CNAME`
   - **Name**: `www`
   - **Value**: `cname.vercel-dns.com`
   - **TTL**: `1 Hour`

4. Збережіть зміни. Оновлення DNS може зайняти від кількох хвилин до 24 годин.

## 2. Як змінювати тексти на сайті

Весь текст на сайті зберігається у двох файлах (для української та англійської мов). Це дозволяє легко редагувати контент без зміни коду.

### Розташування файлів:
- **Українська версія**: `src/lib/messages/ua.json`
- **Англійська версія**: `src/lib/messages/en.json`

### Як редагувати:
1. Відкрийте файл `src/lib/messages/ua.json` (або `en.json`).
2. Знайдіть потрібний текст. Файл структурований по секціях (наприклад, `"home"`, `"footer"`, `"aboutPage"`).
   - Наприклад, щоб змінити заголовок на головній, шукайте в секції `"home"`.
3. Змініть текст у лапках праворуч від ключа.
   - **Було**: `"title": "Старий заголовок"`
   - **Стало**: `"title": "Новий заголовок"`
4. **Важливо**: Не змінюйте ключі (слова зліва), тільки значення (слова справа). Слідкуйте за комами та лапками, щоб не порушити формат JSON.

## 3. Налаштування пошти (Resend)

Сайт використовує сервіс **Resend** для відправки листів.

### Де змінити пошту відправника та отримувача:
Налаштування знаходяться у змінних середовища (Environment Variables).

1. **Локально (на комп'ютері)**:
   - Відкрийте файл `.env` (або створіть його на основі `.env.example`).
   - Знайдіть та відредагуйте:
     ```env
     EMAIL_FROM=contact@onecompany.com    # Пошта, з якої приходять листи (має бути підтверджена в Resend)
     EMAIL_AUTO=auto@onecompany.com       # Куди приходять заявки по Авто
     EMAIL_MOTO=moto@onecompany.com       # Куди приходять заявки по Мото
     RESEND_API_KEY=re_123...             # Ваш API ключ від Resend
     ```

2. **На Vercel (Продакшн)**:
   - Зайдіть у проєкт на Vercel -> **Settings** -> **Environment Variables**.
   - Знайдіть змінні `EMAIL_FROM`, `EMAIL_AUTO`, `EMAIL_MOTO`.
   - Натисніть **Edit**, введіть нові значення та збережіть.
   - Після зміни змінних потрібно зробити **Redeploy** (перезапустити деплой), щоб зміни вступили в силу.

### Важливо про Resend:
Щоб відправляти листи з `contact@onecompany.com`, ви повинні підтвердити домен `onecompany.com` у панелі керування Resend (додати DNS записи TXT, які вони нададуть, у GoDaddy).

## 4. Підв'язка Google Workspace (Корпоративна пошта)

Щоб користуватися поштою Gmail на вашому домені (наприклад, `info@onecompany.com`), потрібно налаштувати MX записи.

1. Зайдіть у **Google Workspace Admin Console**.
2. Почніть процес підтвердження домену. Google надасть вам **TXT запис** або **MX записи**.
3. Зайдіть у **GoDaddy DNS** (там само, де налаштовували домен для Vercel).
4. Додайте **MX записи** від Google:
   - **Type**: `MX`
   - **Name**: `@`
   - **Value**: `SMTP.GOOGLE.COM.`
   - **Priority**: `1`
   (Google може дати 5 записів або один спрощений, використовуйте те, що вони дають).
5. Видаліть старі MX записи, якщо вони є.
6. Додайте **TXT запис** (SPF), щоб листи не потрапляли в спам:
   - **Type**: `TXT`
   - **Name**: `@`
   - **Value**: `v=spf1 include:_spf.google.com ~all`

## 5. Інструкція з Деплою (Розгортання)

### Варіант 1: Автоматичний (через Git) - Рекомендовано
Якщо ваш проєкт підключено до GitHub/GitLab:
1. Зробіть зміни в коді.
2. Зробіть коміт і пуш:
   ```bash
   git add .
   git commit -m "Опис змін"
   git push
   ```
3. Vercel автоматично побачить зміни і почне деплой.

### Варіант 2: Ручний (через термінал)
Якщо ви хочете запустити деплой вручну з комп'ютера:
1. Відкрийте термінал у папці проєкту.
2. Запустіть команду:
   ```bash
   npx vercel --prod
   ```
3. Натисніть `Enter`, якщо запитає підтвердження.
4. Чекайте завершення. Посилання на готовий сайт з'явиться в терміналі.

### Перевірка перед деплоєм
Перед деплоєм корисно запустити перевірку, щоб переконатися, що немає помилок:
```bash
npm run build
```
Якщо ця команда завершиться успішно, деплой теж пройде добре.
