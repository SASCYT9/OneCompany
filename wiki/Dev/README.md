# Dev

Dev notes — інциденти, відлагодження, ідеї, що не належать до жодного конкретного бренду.

## Що сюди писати
- Прод-інциденти (з датою, симптомом, root cause, fix).
- Гідрі-баги (Next.js, Prisma, etc.) — як знайшли і як обійшли.
- Ідеї та TODO, що не вписалися в конкретний task.
- Підказки по інфраструктурі (Vercel, Blob, БД).

## Шаблон інциденту
Файл `wiki/Dev/Incident-YYYY-MM-DD-<short-name>.md`:
- **Симптом** — що зламалось.
- **Виявлено** — як помітили.
- **Root cause** — що було не так.
- **Fix** — commit / PR.
- **Prevention** — що зробили, щоб не повторилось.

Tag: `#bug` або `#incident`.
