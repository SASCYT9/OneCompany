# 🌍 Локалізація, SEO та AI

Глобальний ринок One Company вимагає ідеальної багатомовної архітектури та сильної присутності у пошукових системах (SEO). Це реалізується за допомогою динамічного роутингу, JSON-словників, і автоматичної генерації контенту штучним інтелектом.

## 1. Архітектура Локалізації (next-intl)

Увесь фронтенд додатку працює під управлінням бібліотеки **`next-intl`**.

- **App Router Integration**: Директорія `src/app/[locale]/` гарантує, що кожна сторінка має локалізований префікс у URL (наприклад: `onecompany.global/en/shop` або `onecompany.global/ua/shop`).
- **Словники (Messages)**: Статичні тексти UI (кнопки, меню, поп-апи, футер) знаходяться в JSON файлах:
  - `src/messages/en.json`
  - `src/messages/ua.json`
- **Fallback**: Якщо переклад для ключа відсутній, `next-intl` поверне або fallback мову (зазвичай англійська), або покаже прямий ключ перекладу в розробці.

## 2. Багатомовна База Даних

Локалізація власне **даних каталогу** не покладається на додаткові таблиці для перекладів (як це робиться в старих системах). Вона використовує паралельні мультиязичні стовпці у моделі `ShopProduct` (Див. [[Database ERD Schema]]):
- `titleEn`, `titleUa`
- `shortDescEn`, `shortDescUa`
- `longDescEn`, `longDescUa`
- `seoTitleEn`, `seoTitleUa`
Спеціальні Server Components вирішують, який атрибут повернути під час HTTP-запиту, базуючись на поточній локалі URL.

Для масового перекладу каталогу та постів реалізовано інтеграцію через DeepL API (`deepl-node`) скриптами у директорії `scripts/`.

## 3. SEO-Модуль на базі Штучного Інтелекту (AI)

Для величезного каталогу автозапчастин ручне написання унікальних Мета-тегів (Titles, Descriptions) є неможливим. На допомогу приходить промпт-інжиніринг на боці сервера.

### Механізм `seo-generate`
В адмін-панелі розташовані утиліти для пакетного проганяння продуктів або категорій через AI.
1. **Route Endpoint**: Адмін-запит направляється на `/api/admin/shop/seo-generate/route.ts`.
2. Провайдер **`@google/generative-ai`** (Google Gemini) приймає на вхід існуючі атрибути товару: Бренд, Назва, Специфікація, Марка авто.
3. Процесор (LLM) генерує ідеальний з точки зору густини ключових слів `seoTitle` (до 60 символів) та `seoDescription` (що продає текст до 155 символів).
4. Запис автоматично зберігається в БД (PostgreSQL).

Цей модуль забезпечує високі показники сніпетів у видачі Google (SERP) для нішевих запчастин без залучення армії копірайтерів.

## 4. Динамічний Sitemap

SEO-стратегія доповнюється автоматичним генеруванням мапи сайту XML (Dynamic Sitemap), яка індексує лише публічні продукти (`isPublished = true`) і підтримує генерацію hreflang посилань для кожної доступної мови (`/en` та `/ua`), забезпечуючи правильне розпізнавання гео-доменів роботами Googlebot.
