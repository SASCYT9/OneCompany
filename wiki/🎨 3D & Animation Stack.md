# 🎨 3D & Animation Stack

Проєкт One Company покладається на потужний стек для створення "Pixel-Perfect" інтерфейсів з глибоким зануренням у 3D та плавною анімацією. Цей стек тісно інтегрований з архітектурою **Next.js App Router**, де більшість цих інструментів використовуються виключно в **Client Components** (`"use client"`).

## 🪐 3D Engine (React Three Fiber)

Ми не пишемо `Three.js` код безпосередньо. Замість цього використовується **React Three Fiber (R3F)** — декларативна екосистема для 3D у React.

- **`@react-three/fiber`**: Основний рендерер. Перетворює React компоненти на Three.js об'єкти (наприклад, `<mesh>`, `<boxGeometry>`, `<meshStandardMaterial>`).
- **`@react-three/drei`**: Величезна колекція корисних хелперів для R3F (контролери камери `OrbitControls`, завантажувачі `useGLTF`, `useTexture`, готові оточення `Environment`, `Stage`).
- **`three`** та **`three-stdlib`**: Базові бібліотеки, що стоять за R3F.

### Фізика та Пост-процесинг
- **`@react-three/rapier`**: Фізичний рушій (RigidBodies, Colliders), який дозволяє 3D об'єктам мати вагу, падати та взаємодіяти.
- **`@react-three/postprocessing`** / **`postprocessing`**: Ефекти (Bloom, Depth of Field, Glitch), що накладаються на 3D сцену після її рендеру.

## ✨ 2D Animations & UI Interactions

Для 2D анімацій та скролу використовується комбінація найкращих у своєму класі бібліотек.

- **`framer-motion`**: Основний інструмент для більшості UI анімацій (поява елементів, переходи між сторінками (Page Transitions), жести).
- **`gsap`** (з `@gsap/react`): Використовується для складних таймлайнів та скрол-анімацій (ScrollTrigger), де Framer Motion не вистачає продуктивності або гнучкості.
- **`animejs`**: Додаткова легка бібліотека для специфічних мікро-анімацій.
- **`lenis`** (`@studio-freight/lenis`): Забезпечує **Smooth Scrolling** (плавний скрол). Працює як обгортка навколо нативного скролу браузера, що дозволяє синхронізувати скрол-події з WebGL/GSAP без затримок.

## 🕹️ Утиліти та Контролери

- **`leva`**: Графічний інтерфейс для дебагу та налаштування змінних у 3D сценах (кольори, освітлення, фізика). З'являється панелька поверх сайту.
- **`camera-controls`**: Потужний контролер камери, більш просунутий ніж стандартний `OrbitControls`.
- **`maath`**: Математична бібліотека (сплайни, вектори), часто використовується разом з R3F для плавних математичних перетворень (`damp`).
- **`@use-gesture/react`**: Обробка складних жестів миші/тачу (drag, pinch, wheel) для прив'язки до анімацій.

## Архітектурні Правила
Згідно з [[Codebase Architecture]]:
1. **Ізоляція**: Всі важкі 3D компоненти повинні бути обгорнуті в `<Suspense>` та завантажуватись через `next/dynamic` (Lazy Loading), щоб не блокувати перший рендер (FCP).
2. **Клієнтські межі**: R3F Canvas та всі хуки анімацій (`useFrame`, `useMotionValue`) працюють ТІЛЬКИ в файлах з директивою `"use client"`.
