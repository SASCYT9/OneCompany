# 🛡️ Безпека та Залежності (Security & Dependencies)

Детальний аудит файлу `package.json` та системних залежностей інфраструктури, що використовується в One Company.

## 1. Основний Стек (The Core)
- **Фреймворк**: Next.js (App Router). Працює як на клієнті, так і на сервері (Node.js/Edge).
- **База Даних**: PostgreSQL через **Prisma ORM** (`@prisma/client`).
- **Стилізація**: Tailwind CSS (`tailwindcss`, `tailwind-merge`, `tailwindcss-animate`) плюс `sass` для специфічних випадків оформлення.

## 2. Безпека та Авторизація (Security)
- **NextAuth (`next-auth`)**: Основний провайдер авторизації для Адмін-панелі та порталу для клієнтів. Використовується разом із кастомним Prisma-адаптером для збереження сесій у БД. Захист роутів відбувається через файли `middleware.ts`.
- **Захист Адмін-Панелі**: Доступ до `/api/admin/*` та `/admin/*` суворо обмежується за ролями (`AdminRole`), перевірка відбувається на рівні Server Actions.
- **Шифрування Паролів**: У базі для покупців та адмінів існують `passwordHash` (опрацьовується через `bcrypt` / `argon2` на рівні Node).

## 3. Системи Оплати та Фінанси
- **Stripe (`stripe`)**: Інтегрований для прийому оплат кредитними картами (кредитні ліміти обробляються через Stripe Checkout). Всі транзакції відстежуються вебхуками у `src/app/api/webhooks/`.
- Файли для роботи з фінансами використовують `Decimal` (Prisma/Postgres) для усунення помилок плаваючої коми JS.

## 4. Штучний Інтелект (AI Modules)
Система дуже активно використовує AI-модулі на етапі білд-скриптів та адміністрування:
- **`@google/generative-ai`**: Використовується для генерації SEO-текстів (Route: `/api/admin/shop/seo-generate`), а також для перекладу каталогів.
- **`deepl-node`**: Інтеграція професійного перекладу (DeepL) для конвертації описів товарів (EN ↔ UA) скриптом `scripts/translate-shop-products-en.js`.
- **Локальний AI (`@google/gemini-cli`, утиліти)**: Система розроблена із підтримкою запуску скриптів для AI пакетних завдань.

## 5. Комунікація та Розсилки
- **Telegram Bot (`grammy` та його екосистема)**: Telegram є ключовим інструментом для взаємодії. Використовуються `conversations`, `menu`, `ratelimiter`, `runner`. Обробка ін-меморі або через вебхуки, запуск відбувається через Node (`scripts/telegram-bot-polling.js`).
- **Електронна Пошта (`resend`, `react-email`)**: Транзакційні листи (чеки, зміна статусу замовлення) надсилаються за допомогою Resend. Листи верстаються на React.

## 6. Багатий Клієнтський Досвід
Клієнтська сторона перенасичена 3D та фізичними двигунцями, що вимагає сильного заліза клієнта або хорошої оптимізації рендеру:
- **3D та WebGL**: `@react-three/fiber`, `@react-three/drei`, `three`.
- **Фізика та Пост-Ефекти**: `@react-three/rapier` (фізика, колізії частинок), `@react-three/postprocessing` (bloom-ефекти).
- **Стейт-менеджмент**: `zustand` (для кошика та глобального стейту), `valtio` (стейт 3D об'єктів).
- **Forms & Validation**: Обробка форм під час чекауту відбувається через `react-hook-form` + валідатори.

## 7. Робочі Скрипти (Scripts)
`package.json` містить багато спеціалізованих скриптів:
- `npm run fetch:logos` (Парсинг та завантаження високорівневих лого брендів з інтернету)
- `npm run test:shop` (Комплексне тестування)
- `npm run seo:structure-check` (Аудит лінків на `depth=4`)
- `npm run sync:instagram-blog` (Затягування постів з Instagram на сторінку "Блог" сайту).

## Потенційні Ризики (Vulnerabilities Radar)
1. **Збільшений розмір бандлу (Bundle Size)**: Через наявність масивних модулів `three.js`, `framer-motion` та 3D компонентів, First Contentful Paint може деградувати, якщо не використовувати `next/dynamic` (Lazy Loading).
2. **Rate Limits**: Telegram polling та парсинг логотипів можуть наразитись на блокування по IP від зовнішніх серверів (використання `p-limit` правильне).
3. **Webhook Security**: Важливо регулярно перевіряти секретні ключі та валідацію `stripe-signature` у вебхуках, щоб уникнути фіктивних підтверджень платежів.
