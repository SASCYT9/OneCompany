# Інструкція з управління сайтом OneCompany

Цей документ описує, як вносити зміни в контент сайту, додавати нові бренди та категорії, а також як очистити проєкт від зайвих файлів.

## 1. Управління контентом

### 1.1. Зміна текстів (Переклади)
Всі текстові написи на сайті (заголовки, кнопки, описи) зберігаються у файлах перекладів.
- **Українська**: `src/lib/messages/ua.json`
- **Англійська**: `src/lib/messages/en.json`

**Як змінити:**
1. Відкрийте відповідний `.json` файл.
2. Знайдіть потрібний ключ (наприклад, `"home.title"`).
3. Змініть текст праворуч від двокрапки.
   ```json
   "title": "Ваш новий текст тут"
   ```
   *Увага: Зберігайте структуру JSON (лапки, коми).*

### 1.2. Додавання/Редагування Брендів
Список брендів та їх характеристики знаходяться у файлі:
`src/lib/brands.ts`

**Як додати новий бренд:**
1. Відкрийте `src/lib/brands.ts`.
2. Знайдіть відповідний список (наприклад, `brandsEurope` або `brandsMoto`).
3. Додайте новий рядок:
   ```typescript
   { name: 'Назва Бренду' },
   ```
4. Прокрутіть вниз до об'єкта `brandMetadata` і додайте опис бренду (країна та підкатегорія):
   ```typescript
   'Назва Бренду': { country: 'Germany', subcategory: 'Exhaust' },
   ```
   *Доступні країни та підкатегорії описані на початку файлу.*

### 1.3. Редагування Категорій
Категорії (наприклад, "Exhaust", "Suspension") налаштовуються у файлі:
`src/lib/categoryData.ts`

**Як змінити:**
1. Відкрийте файл.
2. Знайдіть потрібний об'єкт категорії.
3. Ви можете змінити:
   - `title`: Назва (ua/en)
   - `description`: Опис
   - `brands`: Список брендів, що відображаються на картці категорії.

### 1.4. Зображення та Відео
Всі медіафайли знаходяться у папці `public/`.
- **Логотипи**: `public/logos/`
- **Відео**: `public/videos/`
- **Зображення**: `public/images/`

**Щоб замінити зображення:**
1. Завантажте новий файл у відповідну папку.
2. Якщо ви замінюєте існуючий файл, назвіть новий файл так само (наприклад, `hero.jpg`) і перезапишіть старий.
3. Якщо це новий файл, вам потрібно знайти в коді (наприклад, у `src/app/[locale]/page.tsx`), де використовується старий файл, і змінити шлях на новий.

## 2. Очищення проєкту (Зайві файли)

У вашому проєкті є файли, які залишилися від попередніх версій або тестів. Їх можна безпечно видалити, щоб звільнити місце і навести лад.

**Можна видаляти:**
- `wordpress/` (Папка) - Стара версія сайту або тема, не використовується Next.js.
- `wp-content/` (Папка) - Те саме.
- `Design/` (Папка) - Вихідні файли дизайну (збережіть собі копію на комп'ютері, якщо потрібно, але для роботи сайту вони не треба).
- `botw_dump.html`, `botw_post_dump.html` - Тимчасові файли.
- `check-messages.sql`, `test-message-insert.sql` - Тестові скрипти бази даних.
- `test-db.js`, `validate-json.js` - Тестові скрипти.
- `create-screenshot.ps1` - Скрипт для скріншотів.

**Не видаляти:**
- `src/` (Весь код сайту)
- `public/` (Картинки та ресурси)
- `prisma/` (База даних)
- `scripts/` (Корисні скрипти для оновлення логотипів)
- `package.json`, `next.config.ts`, `tsconfig.json` (Налаштування)

## 3. Деплой (Оновлення сайту)

Детальна інструкція з деплою знаходиться у файлі:
`DEPLOYMENT_GUIDE.md`

Коротко:
1. Внесіть зміни.
2. Запустіть `npm run build` для перевірки помилок.
3. Запустіть `git push` (якщо налаштовано) або `vercel --prod` для деплою.
